# 项目交接手册

## 核心设计规范与 UI 说明

为了保证博客最终版本的加载性能与极简视觉，**HZZZZ小站没有使用任何额外的自定义 CSS 代码，也去除了所有“毛玻璃”（frosted glass）UI 特效，Twikoo 评论框需保持原生干净状态。**在后续任何的维护和功能迭代中，请继续保持这一极简的设计理念，避免引入臃肿的样式覆盖。

## 最佳实践：Twikoo 评论系统出海部署教程 (免备案方案)

### 项目背景
* **博客环境**：Hexo 框架 + Stellar 主题。
* **核心痛点**：此前评论系统的后端部署在国内云服务器上，因域名未备案遭遇阿里云严格拦截和阻断。
* **解决思路**：全面出海，白嫖海外 Serverless 算力与免费数据库，彻底绕过 ICP 备案检测，实现零成本、高可用、免备案的评论服务。

---

### 第一步：数据层 - 白嫖 MongoDB Atlas
评论数据需要一个安全可靠的数据库来持久化存储，我们选择 MongoDB 官方提供的云服务。

1. **注册与创建集群**：前往 [MongoDB Atlas](https://www.mongodb.com/cloud/atlas/register) 注册账号，创建一个 `M0 Free` 级别的免费集群（强烈推荐选择 **东京/Tokyo 节点**，国内访问延迟最低）。
2. **配置网络访问白名单**：在左侧菜单进入 `Network Access`，点击 `Add IP Address`，选择 `Allow Access From Anywhere`（即添加 IP 白名单 `0.0.0.0/0`），允许全网访问（因为后续的 Serverless 服务 IP 是动态的）。
3. **获取连接字符串**：在 `Database` 界面点击 `Connect` -> `Connect your application`。复制提供的连接字符串，并将其中的 `<password>` 替换为你的真实数据库密码。
   * **注意**：请确保连接字符串中包含具体的数据库名（例如形如 `mongodb+srv://<user>:<password>@cluster0.xxx.mongodb.net/twikoo?retryWrites=true&w=majority`，其中的 `/twikoo?` 指定了数据库名称）。这串字符就是后续部署逻辑层所需的终极密钥 **`MONGODB_URI`**。

### 第二步：逻辑层 - Netlify 一键部署 (避坑指南)
负责处理评论 API 请求的后端逻辑。市面上有很多 Serverless 平台，但坑也很多。

* **排坑记录**：
  * ❌ **Vercel**：容易不小心触发收费团队（Team）的误导陷阱。
  * ❌ **Hugging Face (HF)**：基于 Docker 的部署经常卡 503 服务不可用，且面临 HF 官方严格的安全扫描误杀。
  * ❌ **Zeabur**：曾经免费的容器部署方案现已转为收费。
* **最终方案（Netlify）**：
  1. 登录 GitHub，Fork Twikoo 官方专为 Netlify 准备的仓库：[twikoojs/twikoo-netlify](https://github.com/twikoojs/twikoo-netlify)。
  2. 登录 [Netlify](https://www.netlify.com/)，点击 `Add new site` -> `Import an existing project`，连接你的 GitHub 账号并选择刚刚 Fork 的 `twikoo-netlify` 仓库。
  3. 在部署设置步骤（Site settings）中，找到 `Environment variables`（环境变量），添加一个键值对：
     * **Key**: `MONGODB_URI`
     * **Value**: `[第一步中获取的数据库连接串]`
  4. 确认无误后点击 `Deploy site`，等待构建流程完毕，日志全绿即代表逻辑层部署成功，Netlify 会为你分配一个默认的 `.netlify.app` 域名。

### 第三步：网络层 - ClouDNS 二级域名极限绕过
为了让评论系统 API 显得更加正规并绕过潜在的网络限制，我们需要绑定自己的自定义域名。

* **痛点**：ClouDNS 等免费域名平台，通常针对根域名（Root Domain）不允许直接配置 CNAME 记录（会与 MX、TXT 等记录冲突）。
* **解决方案**：
  1. 放弃直接使用根域名（如 `hz-artalk.abrdns.com`），改用二级域名 `twikoo.hz-artalk.abrdns.com` 来承载评论服务。
  2. 登录 ClouDNS 控制台，进入你的域名管理面板，添加一条 **CNAME 记录**：
     * 主机名（Host）：`twikoo`
     * 指向（Points to）：填写 Netlify 分配给你的 `xxx.netlify.app` 后缀地址。
  3. 回到 Netlify 控制台，进入 `Domain management`（域名管理），点击 `Add custom domain`，输入你的自定义域名 `twikoo.hz-artalk.abrdns.com`。
  4. 验证 DNS 后，Netlify 会自动为你配置 Let's Encrypt SSL 证书，等待证书状态变绿（开启 HTTPS），网络层配置即大功告成！

### 第四步：展示层 - Hexo 本地配置接入
后端接口准备就绪后，最后一步是让博客前端接入 Twikoo 评论框。

1. 打开 Hexo 项目根目录下的主题配置文件 `_config.stellar.yml`。
2. 定位到 `comments` 配置块，确保 `service` 已经设置为 `twikoo`。
3. 找到底部的 `twikoo` 节点，填入配置好的免备案域名（带 `https://`）：

```yaml
######## Comments ########
comments:
  service: twikoo
  # ... 其他配置 ...

  twikoo:
    envId: 'https://twikoo.hz-artalk.abrdns.com'
```

最后，在命令行执行 `hexo clean && hexo generate`，再部署到你的博客托管平台（如 GitHub Pages / Cloudflare Pages），至此，Twikoo 评论系统便已完美上线！
